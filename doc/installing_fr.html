La documentation est distribuée séparément depuis la version 0.30. C'est un miroir du site web, à l'exception des galeries et des captures d'écran, qui sont des liens externes vers le site. Les versions anglaise et française sont comprises dans l'archive. Si vous avez un lien Internet permanent, vous pouvez initialiser la variable "doc_dir" du fichier geomorphrc avec un lien vers le site web, plutôt que de la télécharger.

Geomorph pour FreeBSD

Dmitry Marakasov a adapté Geomorph pour FreeBSD, en commençant par la version 0.31. Vous pouvez le télécharger ici.


Télécharger POV-Ray

POV-Ray n'est pas obligatoire, mais c'est un "must" si vous désirez voir autre chose que l'aperçu par défaut. Même si vous n'êtes pas familier avec ce logiciel, vous pouvez exécuter les scènes fournies en exemple avec Geomorph.

POV-Ray est un logiciel gratuit, mais non Open source dans sa version courante. Voir la license sur www.povray.org. S'il ne se trouve pas sur votre distribution Linux, vous pouvez le télécharger ici en format .tgz. Sur Ubuntu, POV-Ray est disponible dans le dépôt "multiverse". Une distribution rpm est disponible pour différentes versions de Mandriva / Mandrake sur le site du Penguin Liberation Front.

Installez POV-Ray avant Geomorph, de cette façon Geomorph pourra détecter le nom de l'exécutable POV-Ray lors de l'installation. Autrement, vous devrez inscrire le nom de l'exécutable POV-Ray dans le fichier de configuration de Geomorph geomorphrc.


Compatibilité logicielle

La version 0.50 de Geomorph est basée sur GTK+2 (v. 2.4 ou plus) et utilise OpenGL (MesaGL) pour l'aperçu 3D.

Le paquetage binaire de Geomorph 0.50 est compilé en mode 32 bits sur Debian 4.0. Il a été testé sur Ubuntu 7.04 et 7.10, Fedora 8 et OpenSuse 10.3.

Le paquetage source de Geomorph 0.50 se compile sur ces distributions, pourvu que les paquetages de développement de GTK+2, MesaGL, libgtkglext et libpng soient installés. Le paquetage (lib)gtkglext-dev n'est en général pas installé et est absent de certains des DVDs des distributions. Il peut cependant être trouvé sur Internet, par exemple en faisant une recherche sur http://rpm.pbone.net.

Sur Ubuntu 7.x, la sélection de libgtkglext1-dev dans Synaptic sélectionne toutes les bibliothèques requises pour la compilation comme dépendances.

Geomorph 0.50 fonctionne en mode 64 bits, mais vous devrez le compilez vous-même.

Certains pilotes de cartes graphiques avec certaines distributions peuvent créer une importante fuite de mémoire reliée à OpenGL et à l'accélération matérielle. Voyez la FAQ pour plus d'explications.

Si vous rencontrez des difficultés à exécuter Geomorph, je suggère que vous le démarriez dans un terminal X afin de vérifier le message d'erreur et me le transmettre éventuellement.


Matériel requis

Les exigences matérielles de Geomorph dépendent principalement de la grandeur des images, du nombre de documents ouverts et du nombre de niveaux d'annulation.

Pour des images de 512x512, la configuration minimale recommandée est la suivante:
  • Un bureau de 1024x768.  Plusieurs dialogues n'entreront pas dans un bureau de dimension verticale inférieure, malgré les boutons permettant de masquer les sous-dialogues.
  • 256 Mo de mémoire. Un environnement graphique récent (~2007-2008) peut exiger davantage.
  • Un processeur de 1 Ghz.
Si la dimension verticale de votre bureau vous rend l'utilisation de Geomorph trop difficile, essayez l'interface avec plusieurs fenêtres (style Gimp) plutôt que l'interface intégrée. Le type d'interface est spécifié dans le fichier geomorphrc.

La mémoire recommandée a été établie en créant 5 documents et en les éditant de façon à remplir 5 niveaux d'annulation. L'utilisation maximale de la mémoire devrait alors se situer entre 70 et 80 Mo. Le nombre de niveaux d'annulation est une option du fichier geomorphrc. 

Lorsque l'accélération matérielle de la carte vidéo est activée, la mémoire totale utilisée dépend du pilote de la carte vidéo et de la distribution. L'édition de 5 documents peut alors exiger de 100 Mo à 260 Mo. Une utilisation de 260 MB implique un système de 384 Mo et plus. Cela ne dépend pas de Geomorph, et heureusement, si vous éprouvez de tels problèmes, l'accélération matérielle peut être désactivée pour Geomorph seulement, sans toucher aux autres applications (voir la FAQ).

L'utlisation de POV-Ray consomme de la mémoire, tant que vous n'avez pas fermé la fenêtre affichant l'image rendue. Sur mon système, avec Povray 3,6, chaque image de 640x480 contenant une image de relief de 512x512 occupe 7,3 Mo.


Installer Geomorph à partir du code source

Les étapes sont les mêmes pour la première installation ou pour une mise à jour.

Si vous avez peu d'espace disque, vous voudrez peut-être désinstaller la version précédente (voir l'étape 8, à réaliser dans l'ancien répertoire d'installation).

1. Ouvrir une console et décompresser l'archive:

tar zxf geomorph-0.5x.tgz

2. Passer dans le répertoire de Geomorph:

cd geomorph-0.5x

3. Compiler le logiciel

./configure
make

4. Sous Ubuntu, installer l'exécutable en mode administrateur de cette façon:

sudo make install
... puis passer à l'étape 7.

Sous les autres distributions, se connecter sous root:

su

5. Installer l'exécutable:

make install

6. Quitter root:

exit

7. S'installer comme utilisateur.

./install-user

Ce script crée un répertoire /home/.../geomorph, un fichier de configuration dans ce répertoire, puis installe des raccourcis sur les bureaux Gnome et KDE.

Notes concernant les mises à jour:
  1. Si un répertoire ~/geomorph/scenes existe et contient déjà les définitions de scènes Povray fournies par Geomorph, ces définitions seront renommées <quelquechose>_old.pov, et la nouvelle version de la définition sera copiée dans le répertoire.
  2. Le fichier "geomorphrc" n'est pas remplacé. Il sera mis à jour avec les nouvelles variables, le cas échéant, lors de la prochaine sauvegarde commandée à partir de son dialogue de mise à jour.

8. "make uninstall" comme "root" permet de désinstaller l'application.  Toutefois, les utilisateurs ne seront pas désinstallés. À partir de la version 0.30, la documentation n'est pas effacée non plus, parce qu'elle est installée séparément.


Installer Geomorph à partir de la distribution binaire

Les étapes sont les mêmes pour la première installation ou pour une mise à jour.

Si vous avez peu d'espace disque, vous voudrez peut-être désinstaller la version précédente (voir l'étape 7, à réaliser dans l'ancien répertoire d'installation).

1. Ouvrir une console et décompresser l'archive:

tar zxf geomorph-0.5x-bin.tgz

2. Passer dans le répertoire de Geomorph:

cd geomorph-0.5x

3. Avec Ubuntu, exécuter le script d'installation de cette façon:

sudo ./install-bin

Passer à l'étape 6.

Avec les autres distributions, se connecter d'abord sous root:

su

4. Installer l'exécutable:

./install-bin

5. Quitter root:

exit

6. S'installer comme utilisateur.

./install-user

Voir les commentaires de l'étape 7 de l'installation du programme source, en particulier si vous faites une mise à jour.

7. "./uninstall" dans le même répertoire désinstalle Geomorph, mais sans toucher les répertoires ni les fichiers de configuration des utilisateurs. À partir de la version 0.30, la documentation n'est pas effacée non plus, parce qu'elle est installée séparément.

Installer la documentation

La documentation des versions 0.30 et ultérieures est disponible séparément, dans une archive nommée geomorph-doc-AAAAMMJJ.tgz.

1. Ouvrir une console et décompresser l'archive:

tar zxf geomorph-doc-AAAAMMJJ.tgz

2. Passer dans le répertoire de la documentation de Geomorph:

cd geomorph-doc-AAAAMMJJ

3. Se connecter sous root:

su

4. Exécutez le script d'installation:

./install-doc

5. Quitter root:

exit

Avec une distribution Ubuntu, les étapes 3, 4 et 5 se résument à:

sudo ./install-doc

Le script d'installation copie les fichiers dans /usr/local/share/geomorph/doc.

Si une version antérieure de la documentation existe dans le répertoire /usr/local/share/geomorph/doc, celle-ci est écrasée. Le script refusera d'écraser la version déjà installée avec une version plus ancienne. Un fichier "VERSION" dans ce répertoire contient la date de la version, et un fichier "stamp" est créé avec la date d'installation.

6. Pour avoir accès à cette documentation à partir du menu de Geomorph, initialisez la variable doc_dir du fichier geomorphrc à /usr/local/share/geomorph/doc/index.html, la première fois que vous installez la documentation d'une version 0.30 ou ultérieure.

Vous pouvez effacer la documentation à partir du gestionnaire de fichiers ou en tapant cette commande, comme "root":

rm -rf /usr/local/share/geomorph/doc

Soyez prudent lorsque vous tapez le nom du répertoire, parce que cette commande ne demande pas de confirmation avant d'expédier vos fichiers dans le néant (-f est l'option "force").
